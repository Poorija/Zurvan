Rules of Engagement (ROE)

Web Application Penetration Test

1. Introduction and Authorization

This document outlines the formal agreement and Rules of Engagement (ROE) for a web application penetration test engagement between QuantumLeap Dynamics ("the Client") and GScapy Security Team ("the Tester"). The Client explicitly authorizes the Tester to perform security testing against the application defined within the scope of this document.

2. Period of Performance

All testing activities will be conducted between:

Start Date & Time: 2025-12-01 10:00 PST

End Date & Time: 2025-12-05 18:00 PST

Testing outside of business hours (9 AM - 6 PM, PST) is not permitted without prior approval.

3. Scope of Testing

3.1. In-Scope Application URLs

The following URLs are authorized for testing:

  • https://app.quantumleap.dev
  • https://staging.quantumleap.dev

3.2. Test Accounts & Credentials

The Client will provide the Tester with credentials for the following user roles:

  • Standard User: user@quantumleap.dev / P@ssw0rd123!
  • Admin User: admin@quantumleap.dev / Adm!n$ecr3tP@ss

3.3. Specific Exclusions

The following activities are explicitly forbidden:

  • Automated vulnerability scanning that could impact application availability (e.g., unthrottled Nikto/Gobuster).
  • Testing of the main marketing site: https://www.quantumleap.dev
  • Modification or destruction of data belonging to other users.

4. Methodology & Focus Areas

The Tester will follow the OWASP Web Security Testing Guide (WSTG). Testing will focus on, but is not limited to, the OWASP Top 10 vulnerabilities. Testing of business logic flaws is in scope.
