3.3. Restrictions

The following activities are explicitly forbidden:

  • Load testing or any action that could be perceived as a Denial of Service (DoS) attack.
  • Testing against production user data. Any objects created should be clearly marked as test data (e.g., "GSCAPY_TEST_OBJECT").
  • Accessing or modifying accounts not explicitly provided for testing.

4. Methodology

The Tester will follow the OWASP API Security Top 10, including tests for Broken Object Level Authorization, Broken User Authentication, Excessive Data Exposure, Lack of Resources & Rate Limiting, and Injection flaws.
